## 这个模块解决什么问题

- 新接入 Agent 不知道先读什么
- 多个模块同时扩充时容易放错位置
- 不同 Agent 写法不统一，导致知识库失控
- 课程资料、研究资料、营销资料容易相互污染

## 与其他模块的关系

- `01-核心概念`：负责概念、学科、文化知识
- `02-商业运营`：负责产品、营销、渠道、转化
- `03-教育内容`：负责课纲、讲义、交付、作业
- `04-合规规范`：负责风险边界与平台规范
- `05-项目资料`：负责主体、排期、项目总控
- `06-角色设定`：负责角色分工与协作设定
- `07-经典书籍`：负责经典阅读与原始文本

---

## 使用原则

- 顶层规则优先于模块局部规则
- 通用规则放这里，专题细则再放到业务模块
- 任何 Agent 接入本仓库，先读这里，再进入具体模块
