# 飞书云盘同步指南

> 使用 lark-cli 实现知识库与飞书云盘的双向同步

---

## 一、前置条件

### 1.1 安装 lark-cli

```bash
npm install -g @larksuite/cli
```

### 1.2 配置 lark-cli

```bash
# 初始化配置
lark-cli config init --app-id <your-app-id> --app-secret <your-app-secret>

# 登录授权
lark-cli auth login --domain drive
```

### 1.3 获取飞书文件夹Token

1. 打开飞书云盘
2. 创建或选择一个文件夹
3. 从URL中获取 `folder_token`
   - URL格式: `https://xxx.feishu.cn/drive/folder/fldcnXXXXXX`
   - Token就是 `fldcnXXXXXX` 部分

---

## 二、配置同步

### 2.1 创建配置文件

创建 `~/.feishu-sync.conf`:

```bash
# 飞书云盘同步配置
FEISHU_FOLDER_TOKEN=fldcnXXXXXX

# 可选：指定同步的本地目录
# KNOWLEDGE_BASE=~/文档/道家知识库01
# HAOZE_BASE=~/文档/haoze
# XUANMEN_BASE=~/文档/xuanmen
```

### 2.2 设置环境变量

```bash
# 临时设置
export FEISHU_FOLDER_TOKEN=fldcnXXXXXX

# 永久设置（添加到 ~/.bashrc 或 ~/.zshrc）
echo 'export FEISHU_FOLDER_TOKEN=fldcnXXXXXX' >> ~/.bashrc
source ~/.bashrc
```

## 三、使用方法

### 3.1 交互式同步

```bash
# 运行交互式同步脚本
~/文档/道家知识库01/scripts/feishu-sync.sh
```

菜单选项：
1. 查看飞书云盘状态
2. 推送道家知识库到飞书
3. 从飞书拉取道家知识库
4. 双向同步道家知识库
5. 推送灏泽知识库到飞书
6. 推送玄门知识库到飞书
7. 创建飞书文件夹结构
8. 全量同步（所有知识库）

### 3.2 命令行同步

```bash
# 推送到飞书（本地 → 飞书）
~/文档/道家知识库01/scripts/feishu-auto-sync.sh push

# 从飞书拉取（飞书 → 本地）
~/文档/道家知识库01/scripts/feishu-auto-sync.sh pull

# 双向同步
~/文档/道家知识库01/scripts/feishu-auto-sync.sh sync
```

### 3.3 直接使用 lark-cli

```bash
# 查看飞书云盘文件
lark-cli drive files list --folder-token <folder-token>

# 推送本地目录到飞书
lark-cli drive +push \
    --folder-token <folder-token> \
    --local-dir ~/文档/道家知识库01 \
    --if-exists skip

# 从飞书拉取到本地
lark-cli drive +pull \
    --folder-token <folder-token> \
    --local-dir ~/文档/道家知识库01 \
    --if-exists skip

# 查看同步状态
lark-cli drive status \
    --folder-token <folder-token> \
    --local-dir ~/文档/道家知识库01
```

## 四、定时同步

### 4.1 设置 cron 定时任务

```bash
# 编辑 crontab
crontab -e

# 添加定时任务（每小时同步一次）
0 * * * * ~/文档/道家知识库01/scripts/feishu-auto-sync.sh push >> ~/文档/道家知识库01/scripts/cron.log 2>&1

# 每天凌晨2点全量同步
0 2 * * * ~/文档/道家知识库01/scripts/feishu-auto-sync.sh sync >> ~/文档/道家知识库01/scripts/cron.log 2>&1
```

### 4.2 使用 Hermes cron 功能

```bash
# 在 Hermes 中设置定时任务
hermes cron add --name "feishu-sync" --schedule "0 * * * *" --command "~/文档/道家知识库01/scripts/feishu-auto-sync.sh push"
```

## 五、同步策略

### 5.1 推荐同步策略

| 场景 | 策略 | 说明 |
|------|------|------|
| 日常工作 | Push | 本地修改推送到飞书 |
| 多设备协作 | Pull → Push | 先拉取飞书更新，再推送本地修改 |
| 定时备份 | Push | 定时推送本地知识库到飞书 |
| 灾难恢复 | Pull | 从飞书恢复到本地 |

### 5.2 冲突处理

- **默认策略**: `--if-exists skip`（跳过已存在的文件）
- **覆盖策略**: `--if-exists overwrite`（覆盖已存在的文件）
- **建议**: 首次同步使用 skip，后续同步根据需要选择

### 5.3 注意事项

1. **文件类型**: 只同步文件，不同步在线文档（docx, sheet等）
2. **目录结构**: 本地目录结构会镜像到飞书
3. **删除操作**: 需要显式使用 `--delete-remote` 或 `--delete-local` 参数
4. **网络环境**: 确保网络通畅，大文件同步可能需要较长时间

## 六、故障排除

### 6.1 认证失败

```bash
# 检查认证状态
lark-cli auth status

# 重新登录
lark-cli auth login --domain drive
```

### 6.2 Token错误

```bash
# 验证Token是否有效
lark-cli drive files list --folder-token <your-token>
```

### 6.3 同步失败

```bash
# 查看详细日志
cat ~/文档/道家知识库01/scripts/feishu-sync.log

# 使用dry-run模式预览
lark-cli drive +push \
    --folder-token <folder-token> \
    --local-dir ~/文档/道家知识库01 \
    --dry-run
```

### 6.4 权限不足

确保飞书应用有以下权限：
- `drive:drive` - 云盘访问
- `drive:file` - 文件操作
- `drive:folder` - 文件夹操作

---

## 七、最佳实践

1. **首次同步**: 先用 `--dry-run` 预览，确认无误后再执行
2. **定期备份**: 设置定时任务，自动推送本地知识库到飞书
3. **多设备协作**: 使用 Pull → Push 策略，避免冲突
4. **监控同步**: 定期检查同步日志，确保同步正常
5. **版本控制**: 重要修改前先手动备份
