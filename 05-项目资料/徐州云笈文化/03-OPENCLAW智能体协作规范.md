# 徐州云笈文化 · OPENCLAW 智能体协作规范

> 让其他智能体能够直接读取项目背景，按统一方法做研究、扩库和营销协作

---

## 一、目标

本规范服务两个核心场景：

1. 让研究型智能体自动做 `DeepResearch`，并把结果整理成可写入 wiki 的知识
2. 让营销型智能体读取项目背景后，快速生成选题、脚本、活动文案和转化素材

## 三、建议调用顺序

### 场景 A：外部信息研究并扩库

1. 读取 `xuzhou-yunji-wiki-research`
2. 先读本地知识库相关文档
3. 使用 `DeepResearch` 搜集外部信息
4. 使用 `Summarize` 提炼核心结论
5. 按知识库结构补充到对应文档

### 场景 B：营销内容生产

1. 读取 `xuzhou-yunji-marketing-ops`
2. 先读本地项目定位、产品、合规与近期研究材料
3. 生成选题、内容、活动或转化物料
4. 发布前再对照合规文档做人审

## 五、研究型智能体的标准输出

研究型智能体每次输出建议包含：

| 输出块 | 要求 |
|------|------|
| **研究主题** | 本次研究要解决什么问题 |
| **来源清单** | 主要来源类别与可信度说明 |
| **核心发现** | 3-7 条高价值结论 |
| **对项目的意义** | 为什么对徐州云笈文化重要 |
| **可写入 wiki 的内容** | 结构化内容，不只是摘要 |
| **建议更新位置** | 应写入哪个目录或文档 |
| **后续待研究问题** | 还缺什么信息 |

---

## 六、营销型智能体的标准输出

营销型智能体每次输出建议包含：

| 输出块 | 要求 |
|------|------|
| **目标平台** | 抖音 / 小红书 / 视频号 / 私域 |
| **目标动作** | 流量 / 种草 / 咨询 / 成交 / 复购 |
| **目标人群** | 谁会看、谁会互动、谁会买 |
| **内容形式** | 短视频 / 图文 / 直播提纲 / 社群文案 |
| **核心观点** | 这次最要表达的一个观点 |
| **行动指令** | 用户看完后要做什么 |
| **合规提示** | 容易踩线的地方 |

## 七、协作边界

### 智能体可以做

- 搜集和整理外部信息
- 总结并扩充知识库
- 生成内容初稿、脚本初稿、表格、清单
- 协助建立台历、题库、FAQ、案例库

### 智能体不能替代

- 法律与财税定夺
- 医疗和重大决策建议
- 最终成交承诺
- 最终品牌负责人拍板
