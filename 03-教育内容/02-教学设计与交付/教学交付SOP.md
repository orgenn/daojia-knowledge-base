# 教学交付SOP

> 课程教学设计与交付的标准作业流程

---

## SOP说明

**SOP目的**：确保课程交付的质量和一致性
**适用范围**：所有线上线下的课程交付
**核心原则**：以学员体验为中心，以学习效果为指标
**更新时间**：2026-04-24

---

## 一、课程设计阶段

### 1.1 课程大纲设计

**设计要素**：

| 要素 | 说明 | 合规要求 |
|------|------|----------|
| 课程定位 | 明确课程在体系中的位置 | 不涉及算命、预测等敏感词 |
| 学习目标 | 设定明确可衡量的学习目标 | 用"理解、掌握、获得"等中性词 |
| 适合人群 | 清晰描述目标学员特征 | 避免夸大或绝对化表述 |
| 学习形式 | 线上/线下、录播/直播、社群/一对一 | 明确交付方式 |
| 课程时长 | 明确学习和实践的时间投入 | 实事求是，不虚标 |

**合规检查**：
- [ ] 不使用"保证"、"包准"等承诺词汇
- [ ] 不涉及医疗、投资等重大决策承诺
- [ ] 添加免责声明和理性观看提醒

### 1.2 内容创作原则

**内容要求**：

1. **去神秘化**：
   - 把"神"变成"规律"
   - 把"天机"变成"观察角度"
   - 把"法力"变成"方法"

2. **去吉凶化**：
   - 把"吉凶"变成"适合/不适合"
   - 把"好运"变成"有利条件"
   - 把"坏运"变成"需要调整的情况"

3. **去恐惧化**：
   - 把"灾难"变成"挑战"
   - 把"问题"变成"特质"
   - 把"危机"变成"调整机会"

**表达规范**：
- 使用"建议"、"适合"、"参考"等中性词汇
- 避免"必须"、"一定"、"注定"等绝对化表达
- 强调"学习是为了更好地理解和应用"

### 1.3 互动设计

**互动类型**：

| 互动类型 | 设计要点 | 合规红线 |
|---------|----------|----------|
| 课程提问 | 问题要开放，引导思考 | 不涉及个人隐私和敏感信息 |
| 案例分享 | 案例要匿名处理 | 不泄露学员个人信息 |
| 答疑互动 | 保持客观中性 | 不做具体预测和承诺 |
| 实践打卡 | 鼓励真诚记录 | 不强制分享隐私信息 |

## 二、课程交付准备阶段

### 2.1 开课前的准备工作

**准备清单**：

| 准备项目 | 具体内容 | 责任人 | 完成时间 |
|---------|----------|--------|----------|
| 课程物料 | 课件、讲义、作业模板 | 教研 | 开课前3天 |
| 技术测试 | 直播间、音视频、网络 | 技术 | 开课前1天 |
| 人员安排 | 主讲、助教、客服 | 运营 | 开课前2天 |
| 合规审核 | 内容、话术、承诺 | 合规 | 开课前2天 |
| 兜底方案 | 设备故障、网络问题、时间调整 | 运营 | 开课前1天 |

### 2.2 学员沟通

**沟通内容**：

1. **开课前提醒**：
   - 课程时间和参与方式
   - 学习准备（心态、时间、环境）
   - 课程预期和学习方法

2. **社群建立**：
   - 群规说明（合规要求、互动规则）
   - 学习氛围引导
   - 答疑时间和方式

3. **资料分发**：
   - 学习资料包
   - 作业模板
   - 参考资源

## 三、课程交付执行阶段

### 3.1 直播/线下授课流程

**标准流程**：

**开场环节（5-10分钟）**：
1. 欢迎学员，介绍课程
2. 宣读合规免责声明
3. 说明学习目标和安排
4. 介绍互动方式和答疑规则

**内容讲授环节**：
1. 按照大纲逐步讲解
2. 每个知识点结合案例
3. 适时互动，保持参与度
4. 控制节奏，确保理解

**实践指导环节**：
1. 演示方法的具体应用
2. 引导学员现场练习
3. 提供反馈和调整建议
4. 鼓励分享和提问

**总结答疑环节（10-15分钟）**：
1. 回顾核心要点
2. 集中答疑（优先解决共性问题）
3. 布置课后作业
4. 说明下次课程安排

**收尾环节**：
1. 再次强调理性学习的重要性
2. 鼓励学员在社群分享
3. 提供后续学习路径建议

### 3.2 互动答疑规范

**答疑原则**：

1. **客观中性**：
   - 不做绝对化的判断
   - 不承诺具体效果
   - 使用"可能"、"建议"等词

2. **聚焦课程内容**：
   - 答疑围绕课程讲的知识点
   - 不扩展到课程外的敏感话题
   - 引导学员回到学习本身

3. **保护隐私**：
   - 不询问学员的敏感信息
   - 回答时不涉及具体个人预测
   - 鼓励学员保持边界

**常见问题处理**：

| 问题类型 | 处理方式 | 标准话术 |
|---------|----------|----------|
| 要求预测未来 | 拒绝，转回学习 | "这个我无法预测。我们可以讨论如何应用课程方法来更好地面对未来。" |
| 要求医疗建议 | 拒绝，建议专业渠道 | "关于健康问题，请咨询专业医师。课程只提供文化视角的参考。" |
| 要求婚姻判断 | 拒绝，转回学习 | "这个我不能判断。我们讨论的是如何用传统文化智慧来理解和改善关系。" |
| 要求财运预测 | 拒绝，转回学习 | "我无法预测财运。课程教的是如何建立良好的财务规划和生活习惯。" |

## 四、课后跟进阶段

### 4.1 作业批改与反馈

**批改原则**：

1. **鼓励为主**：
   - 重视学员的真诚分享
   - 发现亮点，给予肯定
   - 提供建设性建议

2. **引导深化**：
   - 指出可以思考的角度
   - 建议实践的调整方向
   - 鼓励持续尝试

3. **保持合规**：
   - 不做具体的命运判断
   - 不承诺练习的必然效果
   - 强调个人差异和持续实践

**反馈模板**：

```
【作业反馈】

✅ 做得好的地方：
- 具体的肯定点1
- 具体的肯定点2

💡 可以思考的角度：
- 引导性的建议1
- 引导性的建议2

🎯 下一步可以尝试：
- 实践建议1
- 实践建议2

⚠️ 温馨提醒：
课程提供的是方法和思路，效果因人而异，需要持续实践和个人调适。
```

### 4.2 学员关怀

**关怀节点**：

| 节点 | 时机 | 内容 |
|------|------|------|
| 学习中途 | 课程进行到1/3时 | 询问学习感受，解答疑虑 |
| 学习收尾 | 课程结束时 | 总结学习收获，提供进阶建议 |
| 实践期 | 课程结束后1-2周 | 询问实践效果，收集反馈 |
| 长期跟进 | 课程结束后1个月 | 了解持续应用情况，推荐后续学习 |

**关怀方式**：
- 社群私信或消息
- 定期直播答疑
- 学员回访电话（重要课程）

## 五、课程评估与优化阶段

### 5.1 学员反馈收集

**收集方式**：

1. **课程满意度调查**：
   - 课程内容满意度
   - 讲师表现满意度
   - 互动体验满意度
   - 整体推荐意愿

2. **开放性问题**：
   - 最有收获的部分
   - 觉得不够清楚的部分
   - 希望补充的内容
   - 对课程的建议

3. **效果跟踪**：
   - 学习后的应用情况
   - 感受到的变化
   - 持续实践的意愿

### 5.2 课程优化迭代

**优化依据**：

1. **学员反馈**：
   - 共同的困惑点：优化讲解方式
   - 不满意的部分：调整内容或表达
   - 希望补充的内容：考虑增加相关内容

2. **完课率数据**：
   - 完课率低的课程：分析原因，优化内容或互动
   - 跳出点高的位置：调整内容节奏

3. **互动数据**：
   - 互动质量低的环节：优化互动设计
   - 答疑集中的问题：补充相关讲解

**优化原则**：
- 小步快跑，持续迭代
- 一次优化一个主要问题
- 优化后测试效果再推广

## 六、线下活动专项SOP

### 6.1 活动前准备

**准备清单**：

| 准备项目 | 具体内容 | 安全要求 |
|---------|----------|----------|
| 场地确认 | 光线、通风、容纳人数 | 消防通道、安全出口 |
| 设备检查 | 音视频、投影、网络 | 设备安全、用电规范 |
| 物料准备 | 讲义、练习材料、茶歇 | 物品安全、过敏提示 |
| 人员安排 | 主讲、助教、安全员 | 人员培训、职责明确 |
| 保险购买 | 活动意外保险 | 保险生效、告知条款 |

### 6.2 活动现场管理

**安全管理**：

1. **入场管理**：
   - 签到登记
   - 免责声明签署
   - 安全须知告知

2. **过程监控**：
   - 观察学员状态
   - 及时处理异常情况
   - 保持通讯畅通

3. **应急处理**：
   - 医疗急救预案
   - 突发情况应对流程
   - 人员疏散预案

## 七、合规风险控制

### 7.1 内容合规检查

**检查清单**：

- [ ] 不使用算命、占卜、改运等、敏感词
- [ ] 不承诺具体效果和结果
- [ ] 不做医疗、投资、婚姻等重大决策建议
- [ ] 添加明确的免责声明
- [ ] 使用"建议"、"参考"等中性词汇

### 7.2 互动合规管控

**管控要点**：

1. **引导学员理性提问**：
   - 明确告知课程不涉及的内容
   - 拒绝回答敏感问题时给出合理解释
   - 引导学员关注学习本身

2. **监控社群讨论**：
   - 及时发现和处理违规言论
   - 引导讨论回到课程主题
   - 鼓励健康的学习氛围

3. **保护学员隐私**：
   - 不在公共场合讨论学员个人情况
   - 不主动询问敏感信息
   - 尊重学员的分享边界

## 八、质量标准

### 8.1 教学质量指标

| 指标 | 目标值 | 测量方式 |
|------|--------|----------|
| 内容合规率 | 100% | 合规审核检查清单 |
| 学员满意度 | ≥4.5/5分 | 满意度调查 |
| 完课率 | ≥60% | 学习数据统计 |
| 互动参与率 | ≥40% | 互动数据统计 |
| 投诉率 | ≤2% | 客服数据统计 |

### 8.2 服务质量标准

**服务承诺**：

1. **响应时间**：
   - 学员提问：24小时内回复
   - 投诉处理：48小时内响应
   - 技术问题：即时响应，及时解决

2. **服务态度**：
   - 耐心细致，不急躁
   - 尊重学员，不居高临下
   - 专业中肯，不随意承诺

3. **问题解决**：
   - 首次解决率 ≥80%
   - 问题跟踪到底，不不了了之
   - 重大问题有升级机制

## 九、团队协作流程

### 9.1 角色分工

| 角色 | 职责 | 协作接口 |
|------|------|----------|
| 主讲讲师 | 课程内容讲授、核心答疑 | 助教、运营 |
| 助教 | 学员辅导、作业批改、日常答疑 | 主讲、运营 |
| 运营 | 课程统筹、学员管理、数据统计 | 主讲、助教、客服 |
| 合规 | 内容审核、风险控制、话术指导 | 所有角色 |
| 技术支持 | 设备保障、故障处理 | 主讲、运营 |

### 9.2 沟通机制

**沟通频率**：

- 开课前：团队准备会
- 课程进行中：实时沟通群
- 每日结束后：当日复盘
- 课程结束后：总结复盘会

**沟通内容**：

- 进度同步
- 问题反馈
- 风险预警
- 调整建议

## 十、SOP维护与更新

### 10.1 更新触发条件

- 出现新的问题或风险
- 收到重要的学员反馈
- 平台规则发生变化
- 有新的课程或服务形式

### 10.2 更新流程

1. 问题收集与分类
2. 解决方案设计与讨论
3. SOP修改与测试
4. 团队培训与推广
5. 效果跟踪与反馈
