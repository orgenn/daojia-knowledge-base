# 徐州云笈文化 Marketing Ops

先读项目背景，再读近期研究，再生成营销内容。

## 先读这些文档

- `05-项目资料/公司信息.md`
- `05-项目资料/徐州云笈文化/01-落地到全面运营节点表.md`
- `05-项目资料/徐州云笈文化/02-运营手册.md`
- `04-合规规范/运营规则总纲.md`
- [references/channel-playbooks.md](references/channel-playbooks.md)

如果任务和外部趋势、政策或竞品有关，先调用 `xuzhou-yunji-wiki-research` 对应输出。

## 适用任务

- 生成月度内容规划、周内容台历
- 生成抖音、小红书、视频号脚本或图文结构
- 生成直播提纲、社群活动文案、私域欢迎语
- 生成产品页文案、活动页卖点、报名转化话术
- 将知识库中的内容转译成用户可理解、可传播的营销表达

## 工作流

### Step 1：锁定任务

先明确：

- 平台
- 目标人群
- 目标动作
- 对应产品或活动

### Step 2：读取本地事实

优先从本地文档中拿信息，不要凭空编品牌故事。重点核对：

- 品牌定位
- 产品结构
- 节点目标
- 合规边界
- 已有研究结论

### Step 3：匹配输出类型

按任务选择输出：

| 任务 | 输出 |
|------|------|
| 选题规划 | 题库、台历、栏目建议 |
| 短视频 | 标题、钩子、口播、CTA |
| 图文种草 | 封面标题、页卡结构、正文逻辑 |
| 直播 | 直播主题、流程、问答、成交段 |
| 私域转化 | 欢迎语、跟进语、标签策略、话术 |
| 活动营销 | 卖点、流程、报名页、预热内容 |

### Step 4：统一表达规则

- 用现代生活语言，不用神秘断语
- 讲问题、讲场景、讲方法，不讲“注定”
- 讲改善路径，不讲夸张结果
- 用“适合/建议/可尝试”，不用“必须/一定/包准”

### Step 5：输出后自检

检查：

- 是否贴合品牌定位
- 是否有明确 CTA
- 是否平台感足够
- 是否踩到合规红线

## 推荐输出骨架

### 选题类

1. 平台
2. 目标人群
3. 选题标题
4. 核心观点
5. 发布建议
6. CTA

### 脚本类

1. 标题
2. 3 秒钩子
3. 主体结构
4. 转化段
5. 风险提醒

### 活动类

1. 活动名称
2. 目标人群
3. 核心卖点
4. 报名路径
5. 预热节奏
6. 转化话术

## 关键约束

- 不要使用医疗承诺
- 不要使用迷信话术
- 不要夸大收益或效果
- 不要引用本地不存在的信息
- 没有事实支撑时，用“建议”“可测试”“待验证”
